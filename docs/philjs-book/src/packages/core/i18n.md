# Internationalization (i18n)

`@philjs/core/i18n` provides locale routing, translation lookup, and formatting helpers.

## Configure the provider

```tsx
import { I18nProvider, type I18nConfig } from '@philjs/core/i18n';

const config: I18nConfig = {
  defaultLocale: 'en',
  locales: ['en', 'es'],
  messages: {
    en: { greeting: 'Hello {{name}}' },
    es: { greeting: 'Hola {{name}}' }
  },
  fallbackLocale: 'en',
  autoDetect: true,
  routePattern: '/[locale]/*'
};

<I18nProvider config={config}>
  <App />
</I18nProvider>
```

## Use translations

```tsx
import { useI18n, useTranslation } from '@philjs/core/i18n';

const { t, formatNumber, formatCurrency } = useI18n();

t('greeting', { vars: { name: 'Phil' } });
formatNumber(12345.67);
formatCurrency(49.99, 'USD');

const translate = useTranslation();
translate('greeting', { vars: { name: 'Phil' } });
```

## Plurals

```tsx
t('items', { count: 2 });
// Use plural rules: zero, one, two, few, many, other
```

## Notes

- `autoDetect` checks the route, localStorage, and browser language.
- `routePattern` lets you model /en/about style routes.
