# Internationalization (i18n) Patterns

Localize PhilJS apps with predictable fallbacks and performance in mind using `@philjs/i18n`.

## Principles

- Extract all user-facing strings; avoid concatenation that breaks grammar.
- Keep locale data lean; lazy-load per locale/namespace.
- Provide sane fallbacks and error handling for missing translations.

## Setup

```tsx
import { createI18n } from '@philjs/i18n';

const i18n = createI18n({
  defaultLocale: 'en',
  fallbackLocale: 'en',
  translations: {
    en: {
      greeting: 'Hello',
      welcome: 'Welcome, {name}!',
      items: { count: 'You have {count} items' },
    },
    es: {
      greeting: 'Hola',
      welcome: 'Bienvenido, {name}!',
    },
  },
});
```

## Basic Usage

```tsx
function Welcome({ name }) {
  // Simple translation
  const greeting = i18n.t('greeting'); // "Hello"

  // With interpolation
  const welcome = i18n.t('welcome', { name }); // "Welcome, John!"

  // Nested keys
  const items = i18n.t('items.count', { count: 5 }); // "You have 5 items"

  return <h1>{welcome}</h1>;
}
```

## Locale Detection

Automatically detect user's preferred locale from multiple sources:

```tsx
import { detectLocale, createLocaleManager } from '@philjs/i18n';

// Detect from URL, cookie, storage, or browser
const locale = detectLocale({
  sources: ['url', 'cookie', 'storage', 'browser'],
  availableLocales: ['en-US', 'es-ES', 'fr-FR'],
  fallback: 'en-US',
});

// Or use the reactive locale manager
const localeManager = createLocaleManager({
  defaultLocale: 'en-US',
  availableLocales: ['en-US', 'es-ES', 'fr-FR'],
  persistTo: ['storage', 'document'],
  detectOnInit: true,
});

// Reactively updates when locale changes
localeManager.setLocale('es-ES');
```

## Number and Currency Formatting

Use built-in Intl-based formatters:

```tsx
import {
  formatNumber,
  formatCurrency,
  formatPercent,
  formatCompact,
  formatBytes,
} from '@philjs/i18n';

formatNumber(1234.56);                    // "1,234.56"
formatNumber(1234.56, { locale: 'de-DE' }); // "1.234,56"

formatCurrency(99.99, 'USD');             // "$99.99"
formatCurrency(99.99, 'EUR', 'de-DE');    // "99,99 €"

formatPercent(0.125, 'en-US', 1);         // "12.5%"

formatCompact(1000000);                   // "1M"
formatCompact(1000, 'en-US', 'long');     // "1 thousand"

formatBytes(1024 * 1024);                 // "1 MB"
```

## Date and Time Formatting

```tsx
import {
  formatDate,
  formatDateStyle,
  formatDateTime,
  formatRelativeTime,
  formatTimeAgo,
} from '@philjs/i18n';

const date = new Date('2024-06-15T14:30:00Z');

formatDateStyle(date, 'full');  // "Saturday, June 15, 2024"
formatDateStyle(date, 'short'); // "6/15/24"

formatDateTime(date, 'medium', 'short'); // "Jun 15, 2024, 2:30 PM"

formatRelativeTime(-1, 'day');  // "yesterday"
formatRelativeTime(3, 'hour');  // "in 3 hours"

formatTimeAgo(new Date(Date.now() - 3600000)); // "1 hour ago"
```

## List Formatting

```tsx
import { formatList } from '@philjs/i18n';

formatList(['Apple', 'Banana', 'Cherry']);
// "Apple, Banana, and Cherry"

formatList(['Red', 'Blue'], 'en-US', 'disjunction');
// "Red or Blue"
```

## Pluralization

```tsx
import { createPlural, pluralRules } from '@philjs/i18n';

const plural = createPlural(pluralRules);

plural('en', 1, ['item', 'items']);   // "item"
plural('en', 5, ['item', 'items']);   // "items"

// French: 0 and 1 are singular
plural('fr', 0, ['article', 'articles']); // "article"
plural('fr', 2, ['article', 'articles']); // "articles"

// Russian has complex rules (one, few, many)
pluralRules.ru(1);  // 'one'
pluralRules.ru(2);  // 'few'
pluralRules.ru(5);  // 'many'
```

## RTL (Right-to-Left) Support

Full support for RTL languages like Arabic, Hebrew, and Persian:

```tsx
import {
  isRtlLanguage,
  getTextDirection,
  setDocumentDirection,
  createRtlConfig,
  wrapInIsolate,
  detectTextDirection,
} from '@philjs/i18n';

// Check if language is RTL
isRtlLanguage('ar');     // true
isRtlLanguage('en');     // false

// Get text direction
getTextDirection('he');  // 'rtl'
getTextDirection('fr');  // 'ltr'

// Set document direction
setDocumentDirection('ar-SA'); // Sets dir="rtl" on <html>

// Create RTL config
const config = createRtlConfig('ar-SA');
// { locale: 'ar-SA', isRtl: true, dir: 'rtl' }

// Wrap mixed content in directional isolate
wrapInIsolate('Hello', 'ltr');   // Wraps in LTR isolate
wrapInIsolate('مرحبا', 'rtl');   // Wraps in RTL isolate

// Detect direction from text content
detectTextDirection('Hello');    // 'ltr'
detectTextDirection('שלום');     // 'rtl'
```

## CSS Logical Properties

Map physical CSS properties to logical equivalents for RTL:

```tsx
import {
  toLogicalProperty,
  flipProperty,
  physicalToLogical,
} from '@philjs/i18n';

// Convert physical to logical
toLogicalProperty('margin-left');   // 'margin-inline-start'
toLogicalProperty('padding-right'); // 'padding-inline-end'

// Flip for RTL
flipProperty('left');               // 'right'
flipProperty('border-top-left-radius'); // 'border-top-right-radius'

// All mappings available
physicalToLogical['width'];  // 'inline-size'
physicalToLogical['height']; // 'block-size'
```

## Display Names

Get localized names for languages, regions, and currencies:

```tsx
import {
  getLanguageName,
  getRegionName,
  getCurrencyName,
} from '@philjs/i18n';

getLanguageName('es');         // "Spanish"
getLanguageName('es', 'de-DE'); // "Spanisch"

getRegionName('US');           // "United States"
getRegionName('DE');           // "Germany"

getCurrencyName('EUR');        // "Euro"
getCurrencyName('JPY');        // "Japanese Yen"
```

## Text Segmentation

Locale-aware word and grapheme counting:

```tsx
import {
  segmentText,
  countWords,
  countGraphemes,
  sortStrings,
} from '@philjs/i18n';

// Segment into words
segmentText('Hello world!', 'word'); // ['Hello', ' ', 'world', '!']

// Count words
countWords('Hello world');           // 2

// Grapheme-correct character count
countGraphemes('Hello');             // 5

// Locale-aware sorting
sortStrings(['cherry', 'apple', 'banana']); // ['apple', 'banana', 'cherry']
```

## Testing

- Snapshot critical pages in multiple locales for layout regressions (rtl/ltr).
- Unit-test formatter outputs.
- Ensure missing keys surface warnings in dev; have fallbacks in prod.

```tsx
import { describe, it, expect } from 'vitest';
import { formatCurrency, isRtlLanguage } from '@philjs/i18n';

describe('i18n', () => {
  it('formats currency correctly', () => {
    expect(formatCurrency(99.99, 'USD')).toMatch(/\$99\.99/);
  });

  it('detects RTL languages', () => {
    expect(isRtlLanguage('ar')).toBe(true);
    expect(isRtlLanguage('en')).toBe(false);
  });
});
```

## Performance Tips

- Avoid shipping all locales; tree-shake unused ones.
- Split locale bundles by route/namespace.
- Use streaming to send base UI fast, then hydrate localized pieces.
- Memoize Intl formatters per locale.
