# Project Structure

A typical PhilJS Rust app keeps UI, server handlers, and assets cleanly separated.

```
my-philjs-app/
├── src/
│   ├── lib.rs              # Shared UI components
│   ├── main.rs             # Server entry (SSR)
│   ├── components/         # Reusable UI
│   ├── routes/             # Route handlers
│   └── server/             # Server-only helpers
├── static/                 # CSS, images, public assets
├── philjs.config.toml      # PhilJS config for Rust projects
├── Cargo.toml
└── package.json            # Node toolchain (generated by cargo-philjs)
```

## Notes

- `lib.rs` defines components and shared UI logic.
- `main.rs` wires the server when using SSR.
- `static/` is served directly by the server integration.
